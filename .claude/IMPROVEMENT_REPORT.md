# MG Mobile App 改善提案レポート

## 📊 優先度別改善一覧

### 🔴 CRITICAL（即座に修正必須・本番環境での実障害リスク）

#### 1. **法人税計算が赤字時も課税される**
- **ファイル**: `src/utils/calculations.js:521-523`
- **問題**: `if (corporateTax <= 7)` で赤字時も強制的に7万円の税を計上
- **影響**: 赤字期間なのに納税義務が発生（会計的に矛盾）
- **修正**:
  ```javascript
  if (profitBeforeTax > 0 && corporateTax < 7) { 
    corporateTax = 7; 
  }
  ```
- **優先度**: ⭐⭐⭐⭐⭐

---

#### 2. **ゼロ除算でNaN発生**
- **ファイル**: `src/utils/calculations.js:200-210`
- **問題**: `totalMaterials / unitCount` で `unitCount === 0` の場合に NaN 計算
- **影響**: 財務計算が破壊される
- **修正**: `if (unitCount === 0) return 0;` チェック追加
- **優先度**: ⭐⭐⭐⭐⭐

---

#### 3. **ルームID・プレイヤーIDの検証なし**
- **ファイル**: `src/App.jsx:100-120`
- **問題**: 任意の入力が許可される→他ユーザーのルームへのアクセス可能（水平権限昇格）
- **影響**: セキュリティ侵害、データ改ざん
- **修正**:
  ```javascript
  const roomIdRegex = /^[a-zA-Z0-9_-]{3,20}$/;
  if (!roomIdRegex.test(roomId)) {
    alert('ルームIDは英数字・アンダースコア・ハイフンのみ');
    return;
  }
  ```
- **優先度**: ⭐⭐⭐⭐⭐

---

#### 4. **Firebase接続エラーが無視される**
- **ファイル**: `src/App.jsx:160-170`
- **問題**: 存在しないルームID でもエラーハンドリングなし
- **影響**: ユーザーが データ未同期に気づかない
- **修正**: `.catch(err => { alert('ルーム接続失敗'); setIsLoggedIn(false); })`
- **優先度**: ⭐⭐⭐⭐

---

#### 5. **数値入力がフィルタリングされていない**
- **ファイル**: `src/components/FinancialStatements.jsx`
- **問題**: `Infinity`, `NaN` などの無効値がそのまま計算に使われる
- **修正**: `if (!Number.isFinite(num)) return 0;`
- **優先度**: ⭐⭐⭐⭐

---

#### 6. **Firebase Security Rules が設定されていない**
- **ファイル**: Firebase コンソール（ Firestore Rules）
- **問題**: 認可ルール（`.read`, `.write`）が明記されていない→全ユーザーが全データ読み取り可能
- **影響**: プレイヤー間のデータ漏洩、改ざん
- **修正**: Firestore Rules を設定（所有者のみアクセス可）
- **優先度**: ⭐⭐⭐⭐⭐

---

### 🟠 HIGH（積極的に修正すべき・ユーザー体験の大幅低下）

#### 7. **Firebase listener の cleanup 漏れ（メモリリーク）**
- **ファイル**: `src/App.jsx:180-195`
- **問題**: `onSnapshot()` を subscribe するが、unmount 時に unsubscribe なし
- **影響**: アプリ使用時間が長いと メモリ使用量が増加し、クラッシュ
- **修正**: `useEffect` に cleanup 関数を追加
  ```javascript
  useEffect(() => {
    const unsub = db.collection(...).onSnapshot(...);
    return () => unsub();
  }, []);
  ```
- **優先度**: ⭐⭐⭐⭐

---

#### 8. **debounce 時間が短すぎる**
- **ファイル**: `src/App.jsx:180-190`
- **問題**: `debounceTime: 5000` (5秒) で素早い更新時にデータロス可能
- **影響**: Firebase 同期が不安定
- **修正**: 10秒に拡大、重複送信チェック追加
- **優先度**: ⭐⭐⭐⭐

---

#### 9. **CashLedger のフルレンダリング**
- **ファイル**: `src/components/CashLedger.jsx:45-65`
- **問題**: フィルタ機能の useEffect で毎回全行を再レンダリング
- **影響**: スクロール時にカクカク、スマホで応答性低下
- **修正**: `useMemo` で `filteredEntries` をメモ化
- **優先度**: ⭐⭐⭐⭐

---

#### 10. **WIP（仕掛品）への完成費計上が会計的に不適切**
- **ファイル**: `src/utils/calculations.js:352`
- **問題**: 「完成費（サ）」が投入前に WIP に含まれている
- **影響**: 仕掛品原価が膨らむ→利益計算が誤る
- **修正**: 会計ルール明確化、計上タイミング調整
- **優先度**: ⭐⭐⭐⭐

---

#### 11. **フォームラベルとアクセシビリティ**
- **ファイル**: 全コンポーネントの `<input>`
- **問題**: `id` 属性がない→`<label>` と紐付けられていない
- **影響**: スクリーンリーダー対応が不完全
- **修正**: `<label htmlFor="input_id">ラベル</label><input id="input_id" />`
- **優先度**: ⭐⭐⭐⭐

---

#### 12. **モーダルのアクセシビリティ**
- **ファイル**: `src/App.jsx:541-614`
- **問題**: `role="dialog"`, `aria-modal="true"` がない、ESC キー対応なし
- **影響**: スクリーンリーダー利用者が操作できない
- **修正**: ARIA 属性追加、ESC キー処理実装
- **優先度**: ⭐⭐⭐⭐

---

#### 13. **ナビゲーションボタンが小さすぎる**
- **ファイル**: `index.css:196-213`
- **問題**: `height: 52px`, `font-size: 0.68rem` でターゲットサイズが小さい
- **影響**: モバイルユーザー（高齢者）が誤クリック
- **修正**: `height: 64px`, `font-size: 0.75rem` に拡大
- **優先度**: ⭐⭐⭐⭐

#### 14. **採用・退職者の重複カウント**
- **ファイル**: `src/utils/calculations.js:255-278`
- **問題**: 同じ人が「採用」→「配置転換」→「退職」で重複カウント
- **影響**: 給与額が不正確
- **修正**: 退職者管理を履歴追跡方式に変更
- **優先度**: ⭐⭐⭐

#### 15. **複数プレイヤーの衝突検出がない**
