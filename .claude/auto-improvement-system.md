# MG Mobile App 自動改善システム

## 概要

複数のAIエージェントが協力して、MGモバイルアプリの改善案を自動分析し、実装する仕組み。

## 改善プロセス

### フェーズ1: 複数エージェントによる改善案分析（並列実行）

1. **Code Quality Agent**: コード品質、バグ、パフォーマンス
2. **UX/UI Agent**: ユーザーインターフェース、操作性、ビジュアル
3. **Functionality Agent**: 機能性、ロジック、計算精度
4. **Security Agent**: セキュリティ、データ保護、入力検証
5. **Architecture Agent**: アーキテクチャ、構造、スケーラビリティ

### フェーズ2: 改善案の統合・優先順位付け

- 全エージェントの提案を集約
- 重複を除去
- CRITICAL → HIGH → MEDIUM → LOW で優先順位付け

### フェーズ3: 修正実装

- CRITICAL/HIGH の問題を自動修正
- 修正後、テスト実行
- Antigravityで動作確認

### フェーズ4: 反復

- 各フェーズで改善が完了したら記録
- 次のサイクルへ

## 実装ガイドライン

### エージェント選択基準

| エージェント | トリガー |
|-----------|---------|
| Code Quality | コード修正・リファクタリング |
| UX/UI | ビジュアル・操作性の改善 |
| Security | 認証・入力・API関連 |
| Functionality | ロジック・計算処理の修正 |
| Architecture | 構造・設計の大幅変更 |

### 修正可能性の判定

修正前に判定:
- [ ] CRITICAL は必ず修正
- [ ] HIGH は積極的に修正
- [ ] MEDIUM は合理的なら修正
- [ ] LOW は将来の改善候補

### 実装時の注意

1. **Antigravityでの検証**: 修正後必ずプレビュー確認
2. **テスト実行**: `npm test` で動作確認
3. **ビルド**: `npm run build` を忘れずに
4. **プッシュ**: `git push` で本番反映

